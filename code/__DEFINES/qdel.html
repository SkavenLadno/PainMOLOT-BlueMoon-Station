code/__DEFINES/qdel.dm

Defines that give qdel hints.

These can be given as a return in /atom/proc/Destroy or by calling /proc/qdel.

QDEL_HINT_QUEUEqdel should queue the object for deletion.
QDEL_HINT_LETMELIVEqdel should let the object live after calling /atom/proc/Destroy.
QDEL_HINT_IWILLGCFunctionally the same as the above. qdel should assume the object will gc on its own, and not check it.
QDEL_HINT_HARDDELQdel should assume this object won't GC, and queue a hard delete using a hard reference.
QDEL_HINT_SOFTFAIL_ALERTQueue normally, but treat a softcheck miss as an operator-facing alert. Prefer this canonical name over the deprecated QUICKDEL alias below.
QDEL_HINT_QUICKDELDeprecated alias kept for compatibility. This does not imply a sub-10s timeout.
QDEL_HINT_SLOWDESTROYKnown-slow cleanup (fires many sub-qdels, events). Suppresses the spurious softcheck warning.
QDEL_HINT_QUEUE_THEN_HARDDELTry soft GC first (Q1 softcheck); on failure, skip warnfail logging and promote directly to hard-delete (Q3).
GC_QUEUE_SOFTCHECKFirst pass: has the object GC'd yet? Most objects pass within 30 seconds.
GC_QUEUE_CHECKBackward compatibility alias for GC_QUEUE_SOFTCHECK.
GC_QUEUE_WARNFAILWarning level: object still alive after 30s. Log, notify admins, continue waiting.
GC_QUEUE_HARDDELETEHard delete: object survived ~2 minutes total. Force del().
GC_QUEUE_COUNTTotal number of queue levels. Increase when adding more.
GC_SOFTCHECK_TIMEOUTTime before softcheck failure is logged (30 seconds).
GC_WARNFAIL_TIMEOUTTime before warnfail escalates to hard delete (90 seconds).
GC_HARDDEL_TIMEOUTTime before hard delete fires (15 seconds).
GC_FILTER_QUEUEHistorical delayed-qdel threshold: timers above this use weakrefs instead of strong refs.
GC_DEL_QUEUEHistorical hard-delete grace window used outside the GC subsystem.
GC_COMPACT_THRESHOLDTrim the dead prefix from a queue once this many tombstoned entries accumulate.
GC_HARDDEL_BUDGET_MIN_MSHOLD-mode millisecond budget for the harddelete pass per fire().
GC_HARDDEL_BUDGET_MAX_MSRECOVER-mode millisecond budget for the harddelete pass per fire().
GC_HARDDEL_OVERFLOW_THRESHOLDQueue depth at which GC enters overflow mode and temporarily spends more time draining hard deletes.
GC_HARDDEL_OVERFLOW_BUDGET_MAX_MSOVERFLOW-mode millisecond budget for the harddelete pass per fire().
GC_HARDDEL_PRESSURE_THRESHOLDQueue depth at which the hard-delete budget starts scaling up from MIN toward MAX.
GC_HARDDEL_HOLD_MAX_PER_FIREHOLD-mode maximum number of del() calls per fire(), regardless of remaining budget.
GC_HARDDEL_MAX_PER_FIRERECOVER-mode maximum number of del() calls per fire(), regardless of remaining budget.
GC_HARDDEL_OVERFLOW_MAX_PER_FIREAbsolute maximum number of del() calls per fire() while overflow mode is active.
GC_HARDDEL_LOBBY_BUDGET_MSLOBBY-mode millisecond budget for the harddelete pass per fire().
GC_HARDDEL_LOBBY_MAX_PER_FIRELOBBY-mode maximum number of del() calls per fire().
GC_HARDDEL_RECOVER_THRESHOLDQueue depth below which GC may enter HOLD mode if the sampled trend is healthy.
GC_HARDDEL_TARGET_Q3_DELTA_PER_SECONDTarget q3 growth rate needed before GC may settle into HOLD mode.
GC_HARDDEL_MODE_HYSTERESIS_SAMPLESConsecutive healthy queue-health samples needed before returning to HOLD mode.
GC_FAILURE_RING_SIZEMax entries in the recent-failures ring buffer.
GC_HARDDEL_RING_SIZEMax entries in the recent-hard-deletes ring buffer.
GC_HARDDEL_YIELD_HISTORY_SIZERolling harddelete yield history window, in GC fire() calls that reached the harddelete pass.
GC_DEPTH_SAMPLE_INTERVALSample queue depths every this many fire() calls (~seconds).
GC_DEPTH_HISTORY_SIZEKeep this many depth samples (GC_DEPTH_SAMPLE_INTERVAL * this = history window in seconds).
