/proc/sanitize_custom_emote_panelRebuilds a custom_emote_panel list with control characters stripped from every emote name (the assoc key) and stored message. Entries whose name becomes empty after cleaning, or collide with an already-cleaned name, are dropped. This both repairs legacy emotes whose control-character names could not round-trip through TGUI (and were therefore impossible to rename or delete) and guards against any such name re-entering the saved data. Always returns a list.
